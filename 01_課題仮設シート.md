# 課題仮説シート

## プロダクト名

gitGUI

## 一言で言うと？

gitのコマンドに応じて視覚的にどういったファイルやブランチの動きをしているかが理解できるツール

## 1. 誰の課題か？（ターゲットユーザー）

- **どんな人？**：git初学者向け
- **人数の規模感**：Gitを学ぶ初級エンジニア全体（継続的に発生する層）を対象(フェルミ推定により約30万人程度と推定)
- **具体的に1人思い浮かべてください**：インターン開始当初の私

## 2. どんな課題か？

- **困っていること**：`git add` / `git commit` / `git push` / `git fetch` のコマンドで何をしているのかがそもそもわからない
- **今どうやって対処しているか**：QiitaやZennなど他者の備忘録を能動的に調べることで学習している
- **なぜそれでは不十分か**：CLI出力は状態の断片しか見えず、5つのエリアを同時に「変化として」把握しづらい。コマンド実行→状態変化の因果がつながりにくく、定着が遅い。実際に手を動かしながら流れを理解したい。

## 3. なぜ今この課題を解くのか？

<!-- 以下のうち当てはまるものがあれば書いてください -->

- エンジニアとしての義務教育：AI駆動開発やチーム開発の前提として、初学者でもGit運用は必須項目として求められる一方、学習初期のつまずきは大きいと考えている
- 自分自身の原体験：インターン初期に「コマンドは打てるが内部状態を説明できない」状態が続き、業務内容との並行学習に苦しんだ
- 既存の解決策が失敗している理由：既存の拡張機能等は内部構成を理解している上で成り立っていると感じるものが多い

## 4. どう解決するか？

- **解決アプローチ（1-2文）**：ブラウザでローカルGitリポジトリを直接読み取り、Working Tree / Staging / Local Branch / Remote Tracking / Remote Branch の5エリアを1秒ポーリングで同期表示する。コマンド名をUI上の矢印として対応づけ、状態遷移を視覚で学べるようにする。
- **モックで見せるコア機能**：リポジトリ選択（`.git`検証付き）→ 5エリア同時表示 → ファイル状態ラベル表示 → ブランチDAGとコミットツールチップ表示 → `git add` / `git commit` / `git push` / `git fetch` に対応した変化の追従
- **ユーザーがこのプロダクトを使ったら何が変わるか**：「とりあえず打つ」から「なぜこの状態になるかを説明できる」に変わり、Git操作のミスや恐怖感を減らせる
